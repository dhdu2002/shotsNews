# shotsNews 작업 요약

## 프로젝트 개요

shotsNews는 Python/PySide6 기반의 Windows 데스크톱 MVP 프로젝트다. 핵심 목표는 다중 소스 이슈를 수집하고, 일별 상위 이슈를 정리해 스크립트 생성 및 동기화 대기열까지 연결되는 실행 파이프라인을 제공하는 것이다.

본 문서는 **현재까지의 구현/검증/저장소 반영 상태**를 정리한 작업 이력 요약이며, 실행 환경 변수와 설치 절차는 `docs/setup.md`에서 별도로 안내한다.

## 완료한 작업

- 로컬 실행 가능한 데스크톱 MVP 파이프라인을 구성했다.
- RSS 우선 수집 및 카테고리 키워드 필터링 흐름을 반영했다.
- Reddit, YouTube(피드 URL), Twitter(X, 선택형) 수집기를 연결했다.
- `MultiSourceCollector` 기반으로 소스별 실패 격리를 적용했다.
- Top-K 랭킹(기본 5건) 처리 경로를 구성했다.
- OpenAI 기반 스크립트 생성 경로를 우선 적용하고, 키 미설정 시 대체 문구 처리 동작을 반영했다.
- 실행 이력/이슈/스크립트/소스 실패/Notion 동기화 큐를 포함하는 SQLite 스키마를 연결했다.
- 자격정보가 없을 때도 안전하게 동작하는 Notion 동기화 분기 처리를 반영했다.
- APScheduler 기반 주기 실행과 수동 실행(`run_now`) 경로를 연결했다.
- PySide6 대시보드 앱에서 런타임 상태 조회(`status`)와 시작/중지 제어(`start`/`stop`) 진입점을 구성했다.
- Windows 실행파일 패키징 기준(PyInstaller)과 실행 확인 절차를 문서화했다.

## 현재 구현 범위

현재 저장소 기준 구현 범위는 아래와 같다.

- 파이프라인 코어: `src/daily_issue_app`
  - 데스크톱 런타임 셸(`DesktopApp`)에서 시작/중지/수동 실행/상태 조회를 제공한다.
  - 스케줄러 시작 및 SQLite 스키마 초기화가 런타임 시작 시 연결된다.
  - 최신 실행 요약, 동기화 대기열, 상위 이슈, 소스 실패 내역을 상태 스냅샷으로 반환한다.
- UI 레이어: `src/ui`
  - PySide6 `QApplication` 생성/재사용, 대시보드 윈도우 생성, 런타임 어댑터 연결, 종료 시 셧다운 훅 연결까지 포함한다.
- 실행/설정/패키징 문서
  - 실행 설정 가이드: `docs/setup.md`
  - 패키징 메모: `packaging/README.md`

본 범위에는 “운영/사업 의사결정용 리서치 결과”가 포함되지 않으며, 해당 항목은 별도 트랙으로 진행 중이다.

## 검증 내역

- MVP 실행 경로 기준 스모크 검증을 완료했다.
- 데스크톱 실행 진입점(`scripts/run_desktop.py`)과 런타임 연결 상태를 확인한 상태다.
- 패키징 기준 실행 확인 절차(실행파일 구동, 대시보드 표시, 로컬 데이터 경로 생성)는 문서화되어 있으며, 최종 스모크 검증 완료 상태로 관리되고 있다.

## 저장소 반영 상태

- 원격 저장소 clone 완료
- 작업 커밋(commit) 완료
- 원격 반영(push) 완료

즉, 현재까지 완료된 MVP 작업과 검증 결과는 저장소 기준으로 반영된 상태다.

## 현재 남은 작업

- 숏폼 수익성/사업성 분석 리서치 진행 중
  - 본 항목은 구현 완료 항목이 아니라, 별도 의사결정 지원을 위한 조사 작업이다.
  - 리서치 결과 정리 및 반영 범위 확정은 후속 단계에서 진행한다.
